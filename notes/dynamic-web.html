Distributing web workload evenly

Just a thought, why is it that pages that load on the browser in modern days are so slow ? Well multiple reasons…

In this post, I will talk about some reasons that I think slow down page perf .

HTTP Archive

This is probably the biggest site to understand the distribtion of content, its bandwidth consumption, and its effect on page load time. Here is a description of bandwidth run for mobiles for 2018.

This is just a small example, the site has lots of good things.

Computing on the client side.

The problem of loading a web page can be distilled into two parts. First, a network constricted resource fetch operation, and second, a compute constricted compute operation.

Web pages designed in the western world, do not represent the average b/w or compute power of people in this world. Therefore for the web to scale, we need newer methods, perhaps, newer more decentralized architectures.

The only way to forever solve this problem is to design web pages by taking into consideration the bandwidth of the user( estimating b/w of the end user is a problem in its own right), there could even be a grade of features and things enabled or disabled based on real time statistics for the given user (this for example is typically done on a video quality degradation).

There are other stop gap solutions, that try to optimize certain aspects of trying to load a page. for example, A data compressing proxy can be used to reduce the strain on end user b/w. Something like a REACT js can be used to more or less stream the changes to the screen, this improves the end user experience, but isn’t that much great on b/w consumption etc etc…

There might still be some scope for translating a given page and its resources based on the end user bandwidth and compute powers on the fly.
